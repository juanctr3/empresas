required
secret

string

required
string

OTP
OTP related APIs

Send OTP
Send a one-time-password to specified mobile number. Requires "otp" API permission.

Request Body schema: multipart/form-data
required
secret
required
string
The API secret you copied from (Tools -> API Keys) page.

type
required
string
Enum: "sms" "whatsapp"
Type of message, it can be SMS or WhatsApp.

message
required
string
OTP message to send, you can use {{otp}} shortcode to include the OTP anywhere in the message.

phone
required
string
Recipient mobile number, it will accept E.164 formatted numbers. Example for Mexico: E.164: +522221234567.

expire	
number
OTP expiration time in seconds. Default value is 300 seconds or 5 minutes.

priority	
number
Default: 2
For WhatsApp only. If you want to send the message as priority, it will be sent immediately. 1 for yes and 2 for no.

account	
string
This is only for WhatsApp type. WhatsApp account you want to use for sending, you can get account unique IDs from /get/wa.accounts or in the dashboard.

mode	
string
Enum: "devices" "credits"
This is only required for SMS type. This is the mode of sending the message. 'devices' allows you to use linked Android devices, while 'credits' uses gateways and requires sufficient credit balance.

device	
string
This is only for SMS type. Linked device unique ID, required for 'devices' mode. You can get linked device unique IDs from /get/devices (Your devices).

gateway	
string or number
This is only for SMS type. Partner device unique ID or gateway ID, required for 'credits' mode. You can get partner device and gateway IDs from /get/rates.

sim	
number
Enum: 1 2
This is only for SMS type. SIM slot number you want to use, for 'devices' mode only.

Responses
200 Successful response
default Error response
Request samples

<?php

// Define the API endpoint and parameters

    "message" => "Your OTP is {{otp}}",
    "phone" => "RECIPIENT_PHONE_NUMBER",
    "expire" => 300
];

// Initialize cURL session

// Set cURL options
curl_setopt($ch, CURLOPT_URL, $url);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, $postFields);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

// Execute the request
$response = curl_exec($ch);
$http_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);

// Handle the response

Send Bulk SMS
Send bulk sms messages. Requires "sms_send_bulk" API permission.

Request Body schema: multipart/form-data
required
secret
required
string
The API secret you copied from (Tools -> API Keys) page.

mode
required
string
Enum: "devices" "credits"
Mode of sending the message, it can be 'devices' which will allow you to use your linked android devices or 'credits' which will allow you to use gateways and partner devices. 'credits' requires you to have enough credit balance to send messages.

campaign
required
string
Name of the campaign, you will see this in the SMS campaign manager.

numbers	
string
List of phone numbers separated by commas. It can be optional if 'groups' parameter is not empty. It will accept E.164 formatted numbers or locally formatted numbers using the country code from your profile settings. E.164: +522221234567 Local: 2221234567.

groups	
string
List of contact group ID's separated by commas. It can be optional if 'numbers' parameter is not empty. You can get group ID's from /get/groups (Your contact groups).

message
required
string
Message you want to send, spintax and shortcodes are supported.

device	
string
Linked device unique ID, this is required if you will send with 'devices' mode. You can get linked device unique ID from /get/devices (Your devices).

gateway	
string or number
Partner device unique ID or gateway ID, this is required if you will send with 'credits' mode. You can get a partner device unique ID and gateway ID from /get/rates.

sim	
number
Enum: 1 2
SIM slot number you want to use. For 'devices' mode only.

priority	
number
Default: 0
Enum: 0 1 2
Priority level. 0 or 1 = high priority (sent immediately), 2 = normal priority (queued). For 'devices' mode only.

shortener	
number
Shortener ID, specify the shortener you want to use if you want to shorten the links in your message. You can get the list of available shorteners from /get/shorteners.

Responses
200 Successful response
default Error response

Request samples

<?php

// Define the API endpoint and parameters

    "message" => "Your message here",
    "numbers" => "PHONE_NUMBERS_COMMA_SEPARATED",
    "sim" => 1
];

// Initialize cURL session
$ch = curl_init();

// Set cURL options
curl_setopt($ch, CURLOPT_URL, $url);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, $postFields);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

// Execute the request
$response = curl_exec($ch);
$http_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);

// Handle the response
if ($http_code === 200) {
    echo "Success: " . $response;
} else {
    echo "Error: HTTP Code " . $http_code . ", Response: " . $response;
}

// Close cURL session

Start WhatsApp Campaign. Requires "start_wa_campaign" API permission.

query Parameters
secret
required
string
The API secret you copied from (Tools -> API Keys) page

campaign
required
number
Campaign ID

Responses
200 Successful response
default Error response
Request simples

<?php

// Define the API endpoint and parameters
$url = "https://whatsapp.smsenlinea.com/api/remote/start.chats?secret=YOUR_API_SECRET&campaign=CAMPAIGN_ID";

// Initialize cURL session
$ch = curl_init();

// Set cURL options
curl_setopt($ch, CURLOPT_URL, $url);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

// Execute the request
$response = curl_exec($ch);
$http_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);

// Handle the response
if ($http_code === 200) {
    echo "Success: " . $response;
} else {
    echo "Error: HTTP Code " . $http_code . ", Response: " . $response;
}

// Close cURL session
curl_close($ch);

{
  "status": 200,
  "message": "WhatsApp campaign has been resumed!",
  "data": false
}

Send USSD Request
Send USSD Request. Requires "ussd" API permission.

Request Body schema: multipart/form-data
required
secret
required
string
The API secret you copied from (Tools -> API Keys) page.

code
required
string
MMI request code. Please make sure that you are using a valid MMI code, if not, it will fail.

sim
required
number
SIM slot number you want to use.

device
required
string
Linked device unique ID. You can get linked device unique ID from /get/devices (Your devices).

Responses
200 Successful response
default Error response

Request samples
<?php

// Define the API endpoint and parameters
$url = "https://whatsapp.smsenlinea.com/api/send/ussd";
$postFields = [
    "secret" => "YOUR_API_SECRET",
    "code" => "*123#",
    "sim" => 1,
    "device" => "DEVICE_UNIQUE_ID"
];

// Initialize cURL session
$ch = curl_init();

// Set cURL options
curl_setopt($ch, CURLOPT_URL, $url);
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, $postFields);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

// Execute the request
$response = curl_exec($ch);
$http_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);

// Handle the response
if ($http_code === 200) {
    echo "Success: " . $response;
} else {
    echo "Error: HTTP Code " . $http_code . ", Response: " . $response;
}

// Close cURL session
curl_close($ch);

  "message": "WhatsApp message has been queued for sending!",
